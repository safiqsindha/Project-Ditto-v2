# Data Provenance — Human-AI Collaborative Sessions

## Source

**Primary source:** SpecStory Markdown files committed to public GitHub repositories  
**Reference methodology:** arXiv 2604.00436 — "Programming by Chat: A Large-Scale Behavioral
Analysis of 11,579 Real-World AI-Assisted IDE Sessions" (March 2026)

SpecStory (https://specstory.com) auto-saves AI coding conversations from Cursor, VS Code +
Copilot, Claude Code, Codex CLI, Gemini CLI, and other AI-assisted IDE tools to
`.specstory/history/*.md` within the developer's repository.  Developers commit these files
as part of their normal workflow, making them publicly and voluntarily accessible on GitHub.

**Note on framing:** These are human-AI *collaborative* sessions, not solo human debugging.
The human drives intent and sub-goal sequencing; the AI generates code in response.  For
Ditto v2's purpose (testing whether human decision sequences under shifting programming
constraints produce detectable structure), this is the appropriate dataset.

## Collection methodology

1. **GitHub Code Search API** with monthly date-prefix partitioning:
   - Query format: `path:.specstory/history filename:YYYY-MM extension:md`
   - Each monthly partition stays well under the 1,000-result API cap
   - Paginated at 100 results/page; up to 10 pages per month-partition
2. **Blob API**: raw file content fetched via `GET /repos/{owner}/{repo}/git/blobs/{sha}`
3. **Deduplication**: by content SHA hash (prevents counting the same file twice)
4. **Secret detection**: patterns for Anthropic/OpenAI keys, GitHub tokens, AWS keys,
   bearer tokens, and credential assignments — any match → reject
5. **Email redaction**: emails replaced with `<redacted-email>` before parsing
6. **License filter**: accept MIT, Apache-2.0, BSD-2/3-Clause, ISC, Unlicense,
   CC0-1.0, MPL-2.0, LGPL — reject GPL, proprietary, unlicensed
7. **Min-event filter**: ≥ 15 parsed events (turns mapped to TrajectoryEvent objects)

## Schema

One trajectory per line in `trajectories.jsonl`.  Same schema as `data/terminal_bench/`
with `"source": "human"`.

Special `args` fields for human sessions:
- `task_transition.args.reason`: first line of the user's request (≤80 chars)
- `file_read.args.source`: "user_paste" (code pasted into chat by developer)
- `file_edit.args.source`: "assistant_codegen" (code generated by AI, applied by developer)
- `error_reveal.args.source`: "user_paste" (error output shared by developer)

## Ethical considerations

1. **Voluntary disclosure**: Developers explicitly committed these files to public repos
   as part of normal git workflows.  There is no ToS violation in indexing public git
   content for research.

2. **License compliance**: Only repositories with permissive open-source licenses are
   included.  GPL repositories are excluded out of an abundance of caution.

3. **Secret / PII filtering**: Every file is scanned for API keys, tokens, and credentials.
   Files containing unredactable secrets are excluded entirely.  Email addresses are
   automatically redacted.

4. **No SpecStory Cloud data**: Only content committed to public GitHub repositories is
   used.  The SpecStory cloud service (cloud.specstory.com) is not accessed.

5. **Research use**: The collected data is used solely for the pre-registered experiment
   described in SPEC.md and will not be redistributed or used for model training.
